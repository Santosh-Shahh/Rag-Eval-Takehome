AetherDB HTTP REST API Reference

This document describes the public HTTP REST API for interacting with AetherDB clusters. All requests must carry content type application/json.

Authentication

All API requests must include the authentication token in the HTTP headers:

X-Aether-Token: bearer_token_here

Endpoints

1. Write Data

Endpoint: POST /api/v1/write

Writes a key-value pair to AetherDB.

Request Body:

{
  "key": "string",
  "value": "string/object",
  "consistency": "quorum"
}

The consistency parameter supports: one, quorum, or all. The default is quorum.

Response:

{
  "status": "success",
  "version": 1042
}

2. Read Data

Endpoint: GET /api/v1/read

Retrieves the value for a given key.

Parameters:

  • key (string, required): The unique key to query.

Response:

{
  "key": "string",
  "value": "string/object",
  "timestamp": 1719700000
}

3. Get Cluster Health

Endpoint: GET /api/v1/health

Returns health metrics of the cluster. Accessing this endpoint does not require authentication headers.
